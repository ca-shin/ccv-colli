## Dati del menu — origine

I dati reali del menu NON erano recuperabili dai backup. Sono stati recuperati dal Supabase del progetto gemello **Camera con Vista** (`pjrdnfbfpogvztfjuxya`), che contiene l'intero sito più la sezione Colli in tabelle con prefisso `colli_`.

- Lettura **sola lettura** delle sole tabelle `colli_*`; nessuna scrittura o modifica sull'origine.
- Mappatura applicata (origine → destinazione):
  - `colli_items` → `dishes`
  - `sort_order` → `order`
  - tabella ponte `colli_item_allergens` → colonna `allergens` (text[]) sui piatti, usando `colli_allergens.source_id` (es. `gluten`, `eggs`)
  - id numerici → id testo con prefisso (`sec_`, `cat_`, `dish_`, `wcat_`, `wine_`); collegamenti preservati
  - scartate colonne non presenti nel nostro schema: `source_id`, `is_active`, `is_available`, timestamp interni
  - regola DNA rispettata: se `allergens` contiene `gluten`, `gluten_free` resta `false`

## Deploy Render — note tecniche (risolte)

- Il build Render con `NODE_ENV=production` fa `npm install` senza devDependencies. `esbuild` (usato da `server:build`) era in devDependencies → errore `esbuild: not found`. Risolto spostando `esbuild` in `dependencies` e usando `npx --yes esbuild` come fallback.
- `dist/` (web export Expo) è versionato di proposito: build Render = `npm install && npm run server:build` (leggero), lo start serve il bundle prebuildato. Rigenerare `dist/` con `npm run web:build` quando cambia il web, poi ricommittare.
- Vulnerabilità npm azzerate (7 → 0) con `npm audit fix`.
